# MedBid Guard 

MedBid Guard 是一个面向医疗设备和医用耗材投标场景的 AI 资格审查工具。本仓库提供一个可直接运行的 MVP，用于在正式投标前对材料进行结构化拆分、资格核验、技术响应检查和格式风险提示。

## 能力

- 上传或粘贴投标相关文本材料，自动形成结构化标书目录
- 检查营业执照、经营资质、产品注册证、厂家授权、售后承诺、类似业绩、技术响应、承诺函等材料是否缺失
- 提示企业名称、产品名称、型号规格、注册证编号、注册人/授权主体等信息是否前后不一致
- 标记授权覆盖不足、证照过期或临近过期、技术参数空项/待补充/疑似负偏离、附件编号不连续等风险
- 输出问题位置、问题类型、风险等级、修改建议和优先处理事项

## 运行方式

```bash
node server.js
```

启动后打开：

```text
http://localhost:3000
```

也可以使用 npm 脚本：

```bash
npm start
```

## 当前规则范围

- 材料完整性：识别核心资格材料和招标要求中明确提到的承诺/业绩类材料
- 信息一致性：企业名称、产品名称、型号、注册证编号、注册人、授权主体
- 有效期：注册证、备案/许可证、授权材料
- 技术响应：空项、待补充、详见附件、疑似负偏离
- 格式附件：附件编号连续性、承诺函日期/落款/签章完整性、正文引用附件缺失

## 已知限制

- 当前 MVP 优先支持文本型材料；扫描版 PDF、图片件、加密文档只会标记为需 OCR/人工复核
- 规则引擎适合预审和风险提示，不替代人工合规审核
- 多产品、多包件场景下的多型号和多注册证，需要结合真实招标范围人工复核
