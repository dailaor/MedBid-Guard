# MedBid Guard

MedBid Guard 是一个面向医疗设备和医用耗材投标场景的 AI 资格审查工具。项目聚焦投标前的材料预检环节，帮助团队在正式递交文件前识别资格缺失、证照不一致、授权覆盖不足、技术响应风险和格式附件问题。

## 项目背景

医疗设备投标材料通常篇幅长、信息分散、附件种类多。资格要求、废标条款、技术参数、授权链路、签章格式和附件编号往往分布在不同章节，投标人员需要在有限时间内反复核对营业执照、经营资质、产品注册证、厂家授权书、技术响应表、承诺函和历史业绩等材料。

在这一流程中，真正高频的风险并不总是复杂判断，而是大量重复且容易遗漏的核验动作，例如：

- 证照类型与拟投产品不匹配
- 产品型号、注册证编号或企业名称前后不一致
- 授权书未覆盖实际投标型号或授权期限临近到期
- 技术响应存在空项、笼统表述或疑似负偏离
- 正文引用了附件，但附件本身缺失或编号不连续

MedBid Guard 的目标不是替代人工决策，而是把这些高频、结构化、可提前发现的问题前置出来，降低投标文件进入最终审阅环节后的返工成本。

## 当前能力

- 上传或粘贴投标相关文本材料，自动形成结构化标书目录
- 检查营业执照、经营资质、产品注册证、厂家授权、售后承诺、类似业绩、技术响应、承诺函等材料是否缺失
- 提示企业名称、产品名称、型号规格、注册证编号、注册人、授权主体等信息是否前后不一致
- 标记授权覆盖不足、证照过期或临近过期、技术参数空项、待补充项和疑似负偏离
- 检查附件编号连续性、承诺函落款/日期/签章完整性，以及正文引用附件缺失等格式问题
- 输出问题位置、问题类型、风险等级、修改建议和优先处理事项

## MVP 范围

当前仓库提供一个可直接运行的 MVP，重点验证以下闭环：

1. 接收投标相关材料
2. 对材料进行基础分类和字段抽取
3. 基于规则进行资格和一致性检查
4. 在配置推理服务后，对技术响应、授权关系和隐性矛盾做模型增强审查
5. 输出结构化目录、风险清单、补充建议和人工复核项

这个版本优先强调可运行性、场景贴合度和结果可解释性，便于后续继续扩展到更复杂的真实招投标材料。

## 模型推理服务接入

项目默认可以在**无模型 key**的情况下直接运行，此时只执行规则引擎审查。  
当配置 OpenAI 兼容推理服务后，后端会在规则结果基础上调用模型，主要用于以下环节：

- 技术响应中的语义负偏离、笼统表述和证明不足判断
- 授权主体、注册人、被授权单位之间的关系解释与冲突提示
- 规则引擎未覆盖的隐性矛盾、材料缺口和人工复核建议生成

配置方式：

1. 复制 `.env.example` 为 `.env`
2. 填写以下环境变量

```text
LLM_API_BASE_URL=https://api.openai.com/v1
LLM_API_KEY=your_api_key_here
LLM_MODEL=gpt-4.1-mini
LLM_TIMEOUT_MS=15000
```

后端会自动读取 `.env`，未配置时会安全回退到规则引擎模式，不影响基础运行。

## 当前规则范围

- 材料完整性：识别核心资格材料和招标要求中明确提到的承诺/业绩类材料
- 信息一致性：企业名称、产品名称、型号、注册证编号、注册人、授权主体
- 有效期：注册证、备案/许可证、授权材料
- 技术响应：空项、待补充、详见附件、疑似负偏离
- 格式附件：附件编号连续性、承诺函日期/落款/签章完整性、正文引用附件缺失

## Roadmap

- 支持 PDF、扫描件和图片材料的 OCR 识别与页级定位
- 引入招标文件章节级拆分，区分资格条款、商务条款、技术条款和废标条款
- 支持多产品、多包件投标场景下的型号映射与证照覆盖判断
- 增加技术参数比对模板，提升参数漏项和负偏离识别精度
- 输出可导出的审查报告，支持问题汇总、定位和补充清单
- 增加可配置规则中心，适配不同医院、集采平台和代理机构的格式要求

## 已知限制

- 当前 MVP 优先支持文本型材料；扫描版 PDF、图片件、加密文档只会标记为需 OCR/人工复核
- 规则引擎适合预审和风险提示，不替代人工合规判断
- 模型增强能力依赖外部推理服务，质量会受到底层模型、上下文裁剪和输入材料质量影响
- 多产品、多包件场景下的多型号和多注册证，需要结合真实招标范围人工复核
